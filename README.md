# RTI Assistant: Government Services RAG Chatbot

> **SMAI Assignment 3 · Theme T10.3 — RTI Filing Helper**

**GitHub:** [https://github.com/Praneethshada/SMAI_A3_RAG](https://github.com/Praneethshada/SMAI_A3_RAG) &nbsp;|&nbsp; **Demo Video (Drive):** [https://drive.google.com/file/d/17KivbFgwDYT864f6cWiiiWzeE496aVZl/view?usp=sharing](https://drive.google.com/file/d/17KivbFgwDYT864f6cWiiiWzeE496aVZl/view?usp=sharing)

A production-quality Retrieval-Augmented Generation (RAG) chatbot that answers citizens' questions about India's Right to Information Act — grounded exclusively in official government PDFs with verifiable source citations.

| File / Folder | Purpose |
|---|---|
| `app.py` | Main Streamlit application — multi-page UI (Home/Resources/Settings), RAG query pipeline, Hindi toggle, source citations, PDF downloads |

| `chroma_db/` | ChromaDB vector index — auto-generated by `ingest.py`, not committed to git |
| `.streamlit/config.toml` | Streamlit server configuration (theme, port) |
| `report.md` | Technical report (Markdown source) |

## Tech Stack

| Layer | Technology | Notes |
|---|---|---|
| Frontend | Streamlit ≥ 1.32 | Multi-page with `st.navigation` |
| Embeddings | `all-MiniLM-L6-v2` | Runs locally on CPU, no API cost |

## Setup & Execution

### 1. Clone and install

Requires **Python 3.12** (see `.python-version`).

```bash
git clone https://github.com/Praneethshada/SMAI_A3_RAG
cd SMAI_A3_RAG
pip install -r requirements.txt
```

### 2. Configure environment

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.3-70b-versatile
```

Get a free API key at [console.groq.com](https://console.groq.com).

### 3. Ingest the PDFs (build vector index)

Run this **once** before starting the app, or whenever you add/change PDFs:

```bash
# First-time ingestion
python ingest.py

# Add new PDFs without re-embedding existing ones (idempotent)
python ingest.py

# Wipe and rebuild from scratch
python ingest.py --reset
```

The script prints a progress summary:
```
[1/4] Loading PDFs...  (88 pages from 5 PDFs)
[2/4] Splitting into chunks...  (539 chunks, avg 536 chars)
[3/4] Enriching metadata...  (chunk_id + doc_type)
[4/4] Embedding with all-MiniLM-L6-v2...
[DONE] Vector store: 'chroma_db' | Total chunks: 539
```

### 4. Run the app

```bash
streamlit run app.py
```

The app opens at `http://localhost:8501` by default.

## Key Features

| Feature | Description |
|---|---|
|**Hindi Language Toggle** | Sidebar/Settings switch — LLM responds in Devanagari script |
|**Multi-turn Conversation** | History-aware condenser resolves follow-up questions correctly |
|**MMR Retrieval** | Diverse, non-redundant context chunks (fetch_k=20, k=6) |
|**Source Citations** | Expandable expanders show exact PDF filename, doc_type, page numbers |
|**PDF Downloads** | Resources page lets users download any knowledge-base PDF |
|**Retry on Rate Limit** | Tenacity: 3 attempts with 2s→4s→8s backoff on Groq 429 |
|**Idempotent Ingestion** | sha256 chunk IDs prevent duplicate embeddings on re-run |
|**doc_type Metadata** | Each chunk tagged Act/Rules/FAQ/Manual |
|**Cached Pipeline** | `@st.cache_resource` — heavy objects loaded once per process |
|**Multi-page Navigation** | Home / Resources / Settings via `st.navigation` |

## ML & System Design Notes

- **Chunk size 700/150**: Tuned for MiniLM's 512-token window. Finer than default 1000 → better recall for specific facts.
- **Section-aware separators**: Splitter tries `Section N` / `Chapter IV` / `Rule N` boundaries first — legal sections stay intact.
- **MMR over top-k**: Prevents 6 near-identical Section 8 excerpts; ensures diverse document types in context.
- **History-aware condenser**: Essential for multi-turn — without it, *"What about the fee?"* retrieves nothing relevant.
- **Stateless design**: All state in `st.session_state` → horizontally scalable, no server-side sessions.
- **Temperature = 0**: Deterministic, factual answers for legal/government information.
